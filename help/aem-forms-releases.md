# AEM [!DNL Forms]リリース{#aem-forms-releases}

Adobe Experience Manager[!DNL Forms]は、AEMにデプロイされるアプリケーションです。 これにより、顧客を引きつける Web およびモバイルエクスペリエンスをエンタープライズフォームとドキュメントにも拡張することができ、ユーザー体験を向上させ、ビジネスの範囲を拡大しながら、複雑なデジタルインタラクションを完成することが可能になります。AEM Quick開始に含まれている標準搭載のAEM [!DNL Forms]には、限られた機能があります。 AEM [!DNL Forms]アドオンパッケージは、[!DNL Adobe Analytics]、[!DNL Adobe Sign]、ドキュメントサービス、ワークフローエンジン、その他多くの機能との統合など、高度な機能を提供します。

>[!NOTE]
>
>AEM Formsは、予定されているAEM Service Packと累積Fix Packのリリース日の1週間後に、アドオンパッケージをリリースします。

次のリストには、今日までにリリースされたAEM [!DNL Forms]アドオンパッケージ、対応するAEMバージョン（前提条件）、パッケージのダウンロードリンク、その他の役立つ情報がすべて含まれています。

## {#things-to-remember}を覚えておくべきこと

* 上記のAEM Formsアドオンパッケージは、OSGiのAEM [!DNL Forms]とJEEのAEM [!DNL Forms]の両方に適用できます。
* AEM [!DNL Forms]アドオンパッケージとQuick Fixはすべて累積的です。つまり、アドオンパッケージとQuick Fixには、対応するバージョンのコンポーネントに対する、前述のすべてのアドオンとQuick Fixが含まれます。 例えば、Quick Fix 2.4.50をインストールする場合、AEM [!DNL Forms]アドオンパッケージ2.4.26が既に含まれています。

   >[!NOTE]
   >
   >これまでのアドオンパッケージとクイックフィックスは、バージョンの最初の2つの番号が同じであることを意味します。

* 常に、利用可能な最新のService Pack/Feature Packアドオンをインストールしてください。 セキュリティ、パフォーマンス、安定性、お客様向けの主要な修正に加え、基本バージョンの一般リリース(GA)以降にリリースされた機能拡張を含むので、強くお勧めします。
* Service Packをインストールするか、以前のバージョンのAEMから新しいバージョンにアップグレードする場合は、必ず対応するAEM [!DNL Forms]アドオンパッケージをインストールしてください。 例えば、AEM 6.1 [!DNL Forms]アドオンパッケージバージョン2.2.4からAEM 6.1 Service Pack 1にアップグレードした場合、AEM 6.1 Service Pack 1をインストールした後にAEM [!DNL Forms]アドオンパッケージバージョン2.4.50をインストールします。
* AEM [!DNL Forms]アドオンパッケージのアンインストールはサポートされていません。
